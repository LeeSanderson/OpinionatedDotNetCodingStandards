## What to build

Add a test (or untestable declaration) for rule **MA0203** — *Do not use return tag for void method* — from the
`Meziantou.Analyzer` analyzer.

## Acceptance criteria

- [ ] Either a `[Fact]` with `[RuleDoc("MA0203", ...)]` exists in
      `tests/Opinionated.DotNet.CodingStandards.Tests/MeziantouAnalyzers/MeziantouAnalyzers3Should.cs`,
      or a class-level `[RuleDoc("MA0203", ..., Untestable = "...")]` exists in `UntestableRules.cs`
- [ ] `RuleDocCoverageShould` passes (no duplicate or missing `[RuleDoc]` entries)
- [ ] If testable: `dotnet test --no-build --filter "FullyQualifiedName~ProhibitReturnTagOnVoidMethod"` passes

## How to implement the test

Add a `[Fact]` to `tests/Opinionated.DotNet.CodingStandards.Tests/MeziantouAnalyzers/MeziantouAnalyzers3Should.cs`:

```csharp
[Fact]
[RuleDoc("MA0203", "Do not use return tag for void method",
    HelpLink = "https://www.meziantou.net/analyzer/rules/203")]
public async Task ProhibitReturnTagOnVoidMethod()
{
    using var project = await CreateProjectBuilderAsync();
    await project.AddFileAsync("Program.cs", """
        namespace test;
        public static class Program
        {
            /// <summary>Does something.</summary>
            /// <returns>Nothing — this is wrong on a void method.</returns>
            public static void DoSomething() { }

            public static int Main() => 0;
        }
        """);
    var buildOutput = await project.BuildAndGetOutputAsync();
    buildOutput.HasError("MA0203").ShouldBeTrue();
}
```

## Confounder playbook (exhaust before marking untestable)

1. **Check EnforceOnBuild** — if the rule is tagged `EnforceOnBuild.Never` in the Roslyn source,
   it is structurally silent at `dotnet build`. Mark untestable and cite the source file.
2. **Check the reporting analyzer's assembly path** — if the reporting class lives in a
   `src/Features/` path (not `src/Analyzers/`), it is IDE/LSP-only. Mark untestable and cite.
3. **Check mscorlib gate** — rules that assert `System.String` is in `mscorlib` cannot fire on
   `net10.0`. Mark untestable with that explanation.

Only move the rule to `UntestableRules.cs` after exhausting all of the above.
